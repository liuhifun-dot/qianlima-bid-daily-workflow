# Phase 4 · 归档、钉盘上传与钉钉发送门禁

## 适用范围

用于把 Phase 3 通过校验的 Excel 日报归档到本地和共享盘，上传到钉盘，并通过钉钉机器人发送日报消息。

本阶段不是“尽量发出去”，而是“所有交付条件满足后再发送”。任何归档或钉盘失败都必须阻止钉钉发送。

## 输入

- Phase 3 校验通过的 Excel 日报。
- Phase 2 最终复判 JSON。
- 本机敏感配置中的 DingTalk Webhook。
- 本机敏感配置中的钉盘目标文件夹。
- 本机敏感配置中的本地归档目录和共享盘目录。

## 输出

- 本地归档文件路径。
- 共享盘归档文件路径。
- 钉盘 `doc_url`。
- 钉钉发送响应。
- Phase 4 日志。

## 执行顺序

必须按以下顺序执行：

1. 本地归档。
2. 共享盘归档。
3. 钉盘上传。
4. 获取 `doc_url`。
5. 生成钉钉消息。
6. 发送钉钉。
7. 记录发送结果。

任何一步失败，都不要执行后续步骤。

## 外部配置路径

本地归档目录从本机配置、`QLM_LOCAL_ARCHIVE` 或显式参数读取：

`<本地归档目录>`

共享盘归档目录从本机配置、`QLM_SHARE_ARCHIVE` 或显式参数读取：

`<共享盘 UNC 归档目录>`

禁止 Agent 临时猜测或替换共享盘地址；只使用预检通过的外部配置。

如果共享盘不可访问，停止，不发送钉钉。

## 文件命名

本地归档建议：

```text
照明招标线索日报_广东_YYYYMMDD_v8.5.xlsx
```

共享盘归档建议：

```text
照明招标线索日报_广东_YYYYMMDD_v1.0.xlsx
```

如果同名文件存在，不覆盖，自动追加版本号：

```text
照明招标线索日报_广东_YYYYMMDD_v8.5_v02.xlsx
```

## 本地归档规则

1. 确认 Phase 3 Excel 已通过校验。
2. 复制到本地归档目录。
3. 复制后检查文件存在、大小一致。
4. 记录本地归档路径。

如果复制失败，停止。

## 共享盘归档规则

1. 检查共享盘路径是否存在。
2. 复制 Excel 到共享盘目录。
3. 复制后检查文件存在、大小一致。
4. 记录共享盘归档路径。

如果共享盘不可达、权限不足或复制失败，停止。

## 钉盘上传规则

钉盘上传必须通过用户已授权的 DingTalk/DWS 工具或运行包中验证过的上传脚本。

上传前检查：

- DWS CLI 可执行。
- 已登录授权。
- OAuth 未过期。
- 目标钉盘文件夹可访问。
- Excel 文件存在。

如果出现以下错误，必须停止：

- OAuth token 过期。
- OSS 签名失败。
- 没有上传权限。
- 找不到钉盘文件夹。
- 上传后没有返回 `doc_url`。

不要在钉盘失败时发送只有本地路径的钉钉消息。

## `doc_url` 门禁

发送钉钉前必须满足：

- `doc_url` 非空。
- `doc_url` 是钉盘或钉钉文档链接。
- 链接对应本次 Excel。
- 本地归档和共享盘归档都成功。

如果没有 `doc_url`，停止并报告：

```text
钉盘上传未完成，缺少 doc_url，禁止发送钉钉。
```

## 钉钉消息格式

使用 markdown 消息。消息要短、清晰、可点击，不要把所有排除项目塞进钉钉。

推荐结构：

```markdown
## YYYY-MM-DD 招投标日报 · QClaw标神

筛选结果：🔥 推荐候选N条 | 📝 待人工复核N条 | 🚫 排除N条

数据源：千里马导出_日期范围（原始N条 → 活跃复判N条）

---

### 🔥 推荐候选（需业务确认）

1. 项目标题 [分数/等级]
   地区 | 预算 | 业务范围 | [千里马链接]

---

### 📝 待人工复核

| 级别 | 标题 | 金额 | 地区 |
|---|---|---|---|
| C | 项目标题 | 金额 | 地区 |

---

### 💡 人工审核建议

真实相关：
1. 项目标题（业务范围）— 地区，金额

假阳性：
- 项目标题（排除原因）

---

### 📁 文件位置

- 钉钉文档：[Excel日报](doc_url)
- 本地归档：本地路径
- 共享盘：共享盘路径
```

## 钉钉消息内容规则

- 标题日期必须是本次日报日期。
- 推荐候选顺序必须与 Excel 一致。
- 待人工复核单独分区。
- 排除只写摘要和典型假阳性，不写 100+ 条长列表。
- 必须包含钉盘链接。
- 可以包含本地路径和共享盘路径，但不能只包含本地路径。
- 不允许乱码、问号、编码异常。
- 不允许写“确认投标”。

## 发送前校验

发送前必须检查：

- Excel 校验通过。
- 本地归档存在。
- 共享盘归档存在。
- 钉盘 `doc_url` 存在。
- 钉钉消息长度合理。
- 消息中没有 `????` 或明显乱码。
- 消息里不含真实密码、Cookie、Token。
- 推荐数量、待复核数量、排除数量与 Phase 2 JSON 和 Excel 一致。

## 发送后校验

发送后必须记录：

- Webhook 响应。
- `errcode`
- `errmsg`
- 发送时间。

只有 `errcode=0` 且 `errmsg=ok` 才算发送成功。

## 常见失败处理

### DWS OAuth 过期

表现：

- OAuth token 过期。
- 登录授权失败。

处理：

1. 停止。
2. 提示用户运行授权命令或重新登录。
3. 不发送钉钉。

### OSS 签名失败

表现：

- OSS signature error。
- 上传失败。

处理：

1. 检查 DWS CLI 版本。
2. 如果是已知版本问题，升级到验证过的版本。
3. 重新上传。
4. 仍失败则停止，不发送钉钉。

### `NODE_OPTIONS` 导致 DWS 失败

表现：

- `--use-system-ca is not allowed in NODE_OPTIONS`

处理：

1. 运行上传命令前清理子进程环境变量 `NODE_OPTIONS`。
2. 重新执行上传。
3. 记录修复动作。

### 共享盘不可达

表现：

- 路径不存在。
- 权限不足。
- 网络不可达。

处理：

1. 停止。
2. 报告共享盘路径。
3. 不上传钉盘，不发送钉钉，除非用户明确改规则。

### 钉钉消息格式不对

表现：

- 内容过长。
- 排除项目刷屏。
- 没有钉盘链接。
- 推荐和待复核混在一起。
- 出现乱码。

处理：

1. 不发送或撤回重发。
2. 使用本文件模板重新生成。
3. 保持推荐顺序与 Excel 一致。

## 成功标准

Phase 4 只有同时满足以下条件才算成功：

- 本地归档成功。
- 共享盘归档成功。
- 钉盘上传成功。
- 获得有效 `doc_url`。
- 钉钉消息格式通过检查。
- Webhook 返回 `errcode=0`。
- 日志记录所有路径和响应。

## 失败输出

如果 Phase 4 失败，必须返回：

- 失败步骤。
- 失败命令或操作。
- 错误原文。
- 已完成的路径。
- 是否已发送钉钉。
- 下一步人工动作。

如果没有发送钉钉，要明确写：

```text
钉钉未发送，原因：缺少 Phase 4 门禁条件。
```

## 排除摘要原因

- 排除摘要按 `second_reason -> reason -> reject_reason -> exclude_reason` 取第一项非空原因。
- 初筛低分原因压缩为“初筛评分不足（总分X），未达推荐阈值”。
- 没有可用原因时不得输出空括号。
