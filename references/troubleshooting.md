# 故障恢复手册

## 适用范围

用于标讯自动化四阶段运行中出现已知故障时恢复执行。先按本文件定位问题，再决定是否继续、重试、降级或停止。

本文件不是流程正文。只在运行失败、页面卡住、Agent 停下来、日志异常、输出不一致时读取。

## 总原则

- 先读最新运行日志和 manifest，再操作页面。
- 先判断当前阶段，不要从头重跑。
- 能恢复的状态自动恢复，不能恢复的状态明确停止。
- 不要清浏览器数据、不要关闭用户正在使用的浏览器、不要删除文件。
- 不要下载旧文件冒充本次结果。
- 不要在 Phase 4 门禁失败时发送钉钉。

## 冷启动恢复顺序

当 Agent 被中断后继续任务：

1. 读取当前运行包路径。
2. 读取最新 `run_manifest`、`pipeline_manifest`、运行日志。
3. 判断最后成功阶段。
4. 如果 Phase 1 已有有效 Excel，用 `--raw-export` 从 Phase 2A 继续。
5. 如果 Phase 2 已有有效复判 JSON，从 Phase 3 继续。
6. 如果 Phase 3 Excel 已校验通过，从 Phase 4 继续。
7. 不要凭聊天记忆猜当前状态。

## Kimi WebBridge stale PID

### 症状

- `running=false`
- `daemon.pid` 存在
- PID 文件中的进程不存在
- 日志出现 `Error: write pid: open daemon.pid: The file exists`

### 原因

Kimi WebBridge 守护进程退出，但旧 PID 锁文件残留。

### 恢复动作

1. 使用正式包或 Skill 中的预检脚本处理 stale PID。
2. 只允许删除 stale PID 锁文件。
3. 删除后重新启动 daemon。
4. 检查：
   - `running=true`
   - `extension_connected=true`
   - 端口为 `10086`

### 自动恢复

`ensure_browser_channel_20260622_v01.py` 已集成自动处理：
- 检测 PID 文件存在但进程不存在时，自动删除
- 自动启动 Chrome（如果未运行）
- 自动启动 daemon
- 等待扩展连接（最多 15 秒）

### 禁止行为

- 不要清 Chrome 数据。
- 不要删除浏览器用户目录。
- 不要重装扩展。

## Kimi WebBridge connected=false

### 症状

- `running=true`
- `extension_connected=false`
- 浏览器扩展未连接。

### 恢复动作

1. 启动或激活已安装 Kimi WebBridge 扩展的 Chrome。
2. 等待扩展连接。
3. 再查状态。
4. 如果仍不连接，记录 Chrome profile 和扩展状态，停止要求人工确认。

### 注意

沙箱可能无法直接启动 GUI Chrome。Kimi WebBridge 依赖用户已运行的 Chrome 和扩展连接，不一定需要 CDP 端口。

## 登录状态超时

### 症状

页面弹窗：

- 登录状态超时，请重新登录
- 请重新登录

或正文读取大量返回 `need_login`。

### 恢复动作

1. 点击弹窗中的 `确定` 或 `确 定`。
2. 从 `https://vip.qianlima.com/` 进入，不优先使用 `/login/`，因为 `/login/` 有时会 403。
3. 如果本机敏感配置提供账号密码，自动登录。
4. 登录后验证会员页或订阅页真实可见。
5. 继续原阶段。

### 禁止行为

- 不要看到 `订阅中心` 就判定已登录。
- 不要跳过正文读取直接生成最终推荐。

## 账号风险提醒

### 症状

弹窗提示密码过于简单或账号风险。

### 恢复动作

1. 点击 `忽略`。
2. 不要点击 `立即修改`。
3. 继续原流程。

## Access Verification 或验证码

### 症状

页面出现访问验证、验证码、安全校验。

### 恢复动作

1. 停止自动化。
2. 截图。
3. 记录当前 URL。
4. 要求人工处理。

### 禁止行为

- 不要绕过验证码。
- 不要暴力刷新。

## 日期控件点击无效

### 症状

- 点击 `近一周` 没反应。
- 下拉不出现。
- Kimi click 无法触发 Vue 日期组件。

### 恢复动作

1. 使用已验证的 Vue 组件方法设置日期。
2. 设置为昨天至今天，不用快捷项。
3. 设置后读取页面显示和内部日期值。
4. 如果显示仍是 `近一周`，重试设置。
5. 重试失败后截图并停止。

### 禁止行为

- 不要改 URL 参数假装设置成功。
- 不要选择 `今天`。

## 导出配置弹窗按钮找不到

### 症状

- 导出配置弹窗出现。
- 找不到 `<button>`。
- 点击 `确定` 后出现 `已订阅成功`，但没有导出记录。

### 原因

千里马的真实导出按钮可能是自定义元素，例如 `div.sure-btn`。

### 恢复动作

1. 在弹窗中查找可见文本：
   - 导出
   - 导 出
   - 确定
   - 确 定
2. 同时查找 class：
   - `sure-btn`
   - primary button
3. 优先点击导出配置弹窗内部的真实导出控件。
4. 点击后检查是否出现导出数量提示或进入导出记录页。

## 导出数量提示

### 症状

弹窗标题或正文包含：

- 导出数量提示
- 访问额度不足
- 是否消耗今日剩余额度
- 继续导出

### 恢复动作

1. 只处理最上层弹窗。
2. 点击 `继续导出`。
3. 如果按钮文字是 `导 出` 或 `导出`，确认它是弹窗主按钮后点击。
4. 等待 15-20 秒。
5. 回到导出记录页读取目标行状态。

### 禁止行为

- 不要点击取消。
- 不要重复提交多个导出任务。

## 导出记录页卡在待生成

### 症状

目标行：

- 状态：待生成
- 操作：生成

### 恢复动作

1. 确认该行是目标行，时间和条数匹配。
2. 如果本次运行尚未提交生成，只点击目标行生成一次；已提交时不得再次点击。
3. 如果出现导出数量提示，解析剩余额度并只点击继续导出一次。
4. 等待 15-20 秒。
5. 重新读取目标行。

## 导出记录页卡在生成中或校验中

### 症状

目标行：

- 状态：生成中 或 校验中
- 操作：刷新

### 恢复动作

1. 等待 10-20 秒。
2. 点击目标行 `刷新`。
3. 重新读取目标行。
4. 重复直到出现 `下载` 或超时。

### 注意

几百条导出通常需要后台校验。不要因为 30 秒没变就直接失败。
Agent 必须让 Phase 1 Python 状态机持续运行并读取心跳，不得终止脚本后自行点击。外层超时至少 25 分钟。

如果进程被系统中断：

1. 找到同次运行的 `qianlima_phase1_checkpoint_*.json`。
2. 执行 Phase 1 脚本并传入 `--resume-context <checkpoint.json>`。
3. 恢复模式只进入导出记录页，沿用原提交时间、目标条数和生成状态。
4. 禁止重新发起导出，也禁止改下方历史行。

## 第一行不是目标行

### 症状

导出记录页第一行可能是：

- 1000 条校验中。
- 另一个导出任务。
- 时间更近但条数不匹配。

### 恢复动作

1. 扫描前 5 行。
2. 首次以本次提交时间和额度确认后的条数锁定目标行；状态和操作列只决定下一步动作。
3. 目标行锁定后不得切换到其他已有下载的行。

### 禁止行为

- 不要默认第一行就是目标。
- 不要因为第一行无下载而失败。
- 不要下载历史可下载行。

## 下载按钮点击失败

### 症状

- 状态已生成。
- 操作列有下载。
- 点击失败或按钮不可见。

### 恢复动作

1. 滚动目标行到视口。
2. 重新获取目标行坐标。
3. 点击目标行下载。
4. 检查下载目录最新文件。

## Phase 2 大量 need_login

### 症状

VIP 正文读取结果中大量项目返回 `need_login`。

### 恢复动作

1. 停止 Phase 2D。
2. 恢复千里马登录态。
3. 重跑正文读取。
4. 不要把正文读取失败的项目推荐为 keep。

## 正文无内容但附件可预览

### 症状

页面正文不足，附件显示 `预览` 或 `下载`。

### 恢复动作

1. 标记 `attachment_required=true`。
2. 先尝试在线预览。
3. 再尝试下载 Word/PDF/Excel。
4. PDF 先文本提取，再 OCR。
5. 仍读不到时，最终只能 `needs_review`。

## Phase 2 校验失败：reject evidence 为空

### 症状

校验报错：

`reject item has empty evidence`

### 原因

初筛排除项没有正文，但最终复判脚本没有把排除原因写入 evidence。

### 恢复动作

1. 对硬排除项目，把硬排除规则和标题证据写入 evidence。
2. 对正文排除项目，把正文证据写入 evidence。
3. 不要跳过校验。

## Phase 3 Excel 格式错误

### 症状

- 没按模板填。
- 全部文字堆在单元格。
- 原始导出数据丢失。
- `needs_review` 混入推荐区。

### 恢复动作

1. 重新读取 `phase3-excel-contract.md`。
2. 基于模板重生成。
3. 保留原始导出数据工作表。
4. 重新跑校验。

## 共享盘失败

### 症状

- 共享盘不可达。
- 路径不存在。
- 被错误改成其他 IP。

### 恢复动作

1. 使用固定路径：
   `<共享盘 UNC 归档目录>`
2. 检查网络和权限。
3. 如果失败，停止 Phase 4。
4. 不发送钉钉。

## DWS 或钉盘失败

### 症状

- OAuth token 过期。
- OSS 签名失败。
- NODE_OPTIONS 报错。
- 全局 dws.cmd 存在但启动失败。
- 上传成功但没有 doc_url。

### 恢复动作

1. 逐个执行候选 DWS 的 --version，跳过损坏命令。
2. 对可用命令执行 drive upload --help。
3. 执行 auth status；失败时停止并请求用户运行 dws auth login。
4. NODE_OPTIONS 报错时，只在 DWS 子进程中清除 NODE_OPTIONS。
5. 上传后解析 doc_url；没有 doc_url 时停止。

### 禁止行为

- 不要因为 dws.cmd 文件存在就判定 CLI 可用。
- 不要在钉盘失败时发送正常钉钉日报。
- 不要只发本地路径。
## 钉钉消息格式错误

### 症状

- 消息太长。
- 排除项目刷屏。
- 没有钉盘链接。
- 出现乱码或问号。
- 写了“确认投标”。

### 恢复动作

1. 重新读取 `phase4-archive-dingtalk.md`。
2. 使用短模板。
3. 推荐、待复核、排除摘要分区。
4. 确保包含 `doc_url`。
5. 再发送。

## Agent 执行闭环失效

### 症状

- Agent 每一步都停下来让用户选择。
- 已有 SOP 但仍反复问同一问题。
- 停止后不会读日志继续。

### 恢复动作

1. 明确当前阶段。
2. 读取最新日志和 manifest。
3. 找到最后成功产物。
4. 从断点继续。
5. 只有遇到硬停止条件才问用户。

## 失败报告格式

失败时必须返回：

- 阶段。
- 页面 URL 或脚本命令。
- 错误原文。
- 已完成产物。
- 未完成产物。
- 是否触发硬停止。
- 下一步恢复动作。

不要只写“失败”或“需人工处理”。

## Kimi 不可用但 Phase 2 需要证据

### 判断顺序

1. 先检查 stale PID 和扩展连接。
2. 检查 CDP 9222 是否已经登录千里马。
3. 如果 CDP 已登录，使用正式 CDP v04 正文附件读取器，不用临时标题读取脚本。
4. 运行 CDP v03 证据校验器。
5. 校验失败时停止 Phase 2D/3/4。

### CDP v04 正常输出

- 每个输入项目都有一条正文输出和一条附件输出。
- 正文至少 200 个有效字符并通过项目身份校验。
- 附件必须记录下载、解析、SHA256 和未完成原因。
- 只读到部分附件时 attachment_read_ok=false。
- 扫描 PDF、图片、RAR/7Z 或外链失败时进入未完成证据。

### 禁止行为

- 不要把只读到标题和元数据的临时脚本当正式证据。
- 不要因为读到一个附件就把整个附件集标成功。
- 不要把 4 个字符或扫描 PDF 空文本标成功。
- 不要在证据校验失败时发送正常日报。
## Unattended Run Failure Alert

### Rule

Unattended automation may stop. A stopped run is acceptable when it prevents a false success report.

### Recovery Action

When stopping unattended, write a failure report containing:

- run_id;
- failed phase;
- blocking condition;
- last successful artifact;
- screenshot or page text if browser-related;
- next human action;
- whether DingTalk was sent.

If a failure-notification channel is configured separately, send a failure alert. Do not send the normal daily report.


## 2026-06-23 冷启动回归新增规则

- 把搜索框 placeholder 中的“请输入”与真实登录表单分开；只有账号框加密码框，或账号框加明确登录按钮，才判定为登录页。
- 下载监控只接受 Excel 或专用 CDP 下载目录中的候选文件；忽略 .crdownload、~$ 和隐藏临时文件。
- 不把“导出Word/Excel/PDF”“供应商报名”“投标人登记申请表”识别成采购证据附件。
- 不把 CSS、JavaScript、HTML 跳转页或未知文件类型标记为附件下载成功。
- 当本次存在必读附件且成功读取数为 0 时，判定附件证据通道整体失败。允许生成 dry-run 草稿，禁止正式归档与钉钉日报。
- 每个 run_id 单独生成报告；失败报告、Phase 1 报告和完整流程报告不得追加到同一个文件形成前后矛盾。
- 钉钉消息只显示中文业务说明，不得出现 attachment_required、attachment_read_ok、BOQ 等内部字段。

## 外部 SPA 附件下载页

当千里马两步下载解析出的真实地址是 `index.html#/downloadFile...` 等前端路由时：

1. 先执行普通 HTTP 下载。
2. 返回 HTML 时，使用 CDP 新建临时标签页，监听下载事件和带文件响应头的网络响应。
3. 单个附件浏览器捕获总等待时间不得超过约 30 秒。
4. 捕获真实文件后按文件头校验并解析；只捕获 HTML、静态资源或空响应时判定未读取。
5. 关闭自己创建的临时标签页，不关闭用户浏览器。
6. 失败后写入 `attachment_required=true`、`attachment_read_ok=false` 和中文原因，交给 Phase 2D 待人工复核；禁止无限重试。
