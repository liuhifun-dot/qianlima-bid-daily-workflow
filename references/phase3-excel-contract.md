# Phase 3 · Excel 日报模板契约

## 适用范围

用于把 Phase 2 最终业务复判 JSON 生成“照明招标线索日报”Excel。目标是让业务人员能直接打开、复核、跟进，而不是生成一个只有数据但格式混乱的表。

本阶段必须优先保证模板格式、信息完整性和人工复盘能力。

## 输入

- Phase 2 最终业务复判 JSON。
- Phase 1 原始导出 Excel。
- 权威 Excel 模板：`照明招标线索日报_格式样板_20260616_v05.xlsx`。
- Skill 只打包 assets 内的 v05 权威模板；prepare_runtime 会把它复制到运行目录的 02_模板。不要引用归档样例。
- 当天运行日期、日期范围、数据来源、导出条数。

## 输出

- 当天 Excel 日报。
- Excel 校验结果。
- 日报生成日志。

## 模板定位

- Skill assets 和 active runtime 的 02_模板各只保留一个权威模板。
- 运行时只能使用 prepare_runtime 清单中的 v05 模板；归档、样例预览和备份模板不能作为生成依据。
- Excel 生成脚本和校验器必须共同指向同一份权威模板。

## 基本原则

- 必须基于模板生成，不能从空白工作簿重建。
- 必须保留模板样式，包括合并单元格、列宽、行高、颜色、边框、换行、冻结窗格。
- 必须在“📋 全部标讯”保留 Phase 1 原始数据行，并追加复判字段供人工复盘。
- 不能删除原始数据行或原始业务字段。
- 不能把缺失信息乱填为推测值；没有就留空或写“未获取”。
- 推荐项目排序必须和钉钉消息排序一致。

## 禁止行为

- 不允许用 pandas 直接重建一个新 Excel 替代模板。
- 不允许把所有信息塞进一个单元格，导致没有换行、没有按格填写。
- 不允许把 `needs_review` 混进推荐候选区。
- 不允许把推荐候选写成“确认投标”。
- 不允许把“📋 全部标讯”缩减成只含入选项目。
- 不允许只生成汇总页，不生成项目详情页。
- 不允许因为字段缺失就编造金额、截止日期、联系人、工期。

## 工作簿结构

推荐结构：

1. 📊 统计
2. 📋 全部标讯
3. 📍 今日推荐标讯
4. `项目1`
5. `项目2`
6. `项目3`
7. 后续项目详情页

如果模板原始 sheet 名不同，保持模板命名，但必须表达同样的信息结构。

## 统计页规则

统计页必须使用业务准确措辞。

允许：

- 总复判
- 推荐候选
- 待人工复核
- 排除
- 数据来源
- 导出条数
- 入选条数

禁止：

- 确认投标
- 已确定项目
- 必中项目
- 已确认可做

示例：

```text
总复判：10 条
推荐候选：2 条
待人工复核：1 条
排除：7 条
数据来源：千里马导出_广东_20260615
```

## 全部复判项目页

每条复判项目一行。必须包含：

- 序号
- 结论：推荐候选 / 待人工复核 / 排除
- 等级：A / B / C / D
- 标题
- 地区
- 金额
- 截止日期
- 业务范围
- 证据摘要
- 风险点
- 原因
- 千里马链接

规则：

- `keep` 显示为“推荐候选”。
- `needs_review` 显示为“待人工复核”。
- `reject` 显示为“排除”。
- 证据摘要必须来自 `evidence`，不能为空。
- 链接必须可点击。

## 今日推荐标讯页

必须分区，不能混排：

1. 推荐候选
2. 待人工复核
3. 排除摘要

### 推荐候选区

只放 `final_decision=keep` 的项目。

每条至少显示：

- 排名
- 标题
- 地区
- 金额
- 等级
- 业务范围
- 推荐依据
- 待确认点
- 链接

文案示例：

```text
B级推荐候选：小额 LED 照明更换，正文显示含安装，需业务确认是否跟进。
```

### 待人工复核区

只放 `final_decision=needs_review` 的项目。

每条至少显示：

- 标题
- 金额
- 地区
- 复核原因
- 缺失证据
- 下一步人工动作
- 链接

示例：

```text
附件未读：页面正文不足，3 个 docx 附件可能包含技术规范，需人工预览确认是否含电气/照明范围。
```

### 排除摘要区

不需要展示全部 100+ 条排除项目的长列表。展示：

- 排除总数
- 主要排除原因统计
- 典型误命中案例

但 `全部复判项目` 页必须保留每条排除项目的结论和原因。

## 项目详情页

每个推荐候选和待人工复核项目应有详情页。详情页必须按模板格子填写，不要堆成长段落。

建议字段：

- 公告类型
- 省份/城市
- 发布日期
- 截止日期
- 预算金额
- 发包方
- 招标代理
- 千里马链接
- 二筛决定
- 二筛原因
- 业务方向
- 分析意见
- 项目进度跟踪
- 招标正文摘要
- 附件下载/预览情况
- 联系方式

填写规则：

- 多条内容用单元格内换行。
- 长正文摘要保留关键段落，不要塞完整正文。
- 证据和判断分开写。
- 未获取的信息留空或写“未获取”，不要猜。

## 原始导出数据保留

权威模板使用“📋 全部标讯”作为原始底表。生成脚本必须：

- 保留 Phase 1 原始 Excel 的全部数据行。
- 不删原始业务字段。
- 在后续列追加复核、评分、分析、排除原因和二筛结果。
- 原始行数应与 Phase 1 Excel 有可解释的表头差异。
- 不得只写入入选项目。

目的：

- 人工复盘 Agent 是否漏筛。
- 调整筛选规则。
- 追查日报项目对应的原始记录。

如果未来模板改为独立“原始导出数据”工作表，也必须满足同样的数据完整性要求。
## 排序规则

推荐候选排序：

1. 业务匹配度高。
2. 证据完整。
3. 金额和跟进价值高。
4. 截止时间更紧急。
5. 地区更匹配。

待人工复核排序：

1. 可能价值高。
2. 缺失证据少。
3. 人工确认成本低。

钉钉消息中的推荐顺序必须与 Excel 推荐候选顺序一致。

## 文件命名

建议：

```text
照明招标线索日报_广东_YYYYMMDD_v8.5.xlsx
```

如果同一天重复生成：

```text
照明招标线索日报_广东_YYYYMMDD_v8.5_v02.xlsx
```

不要覆盖已归档文件。

## 校验规则

生成后必须校验：

- 文件存在。
- 文件可被 openpyxl 打开。
- 必要 sheet 存在。
- 统计数量与 Phase 2 JSON 一致。
- 推荐候选数量一致。
- 待人工复核数量一致。
- 排除数量一致。
- “📋 全部标讯”保留全部原始数据行，并包含可追溯的复判字段。
- 推荐候选不包含 `needs_review`。
- `needs_review` 不混入推荐候选区。
- 不出现“确认投标”。
- 关键链接可点击。

## 失败处理

如果校验失败：

1. 不进入 Phase 4。
2. 记录失败项。
3. 修正生成脚本或输入数据。
4. 重新生成 Excel。
5. 再次校验。

不要跳过模板校验，也不要把格式错误的 Excel 发到钉钉。

## 成功标准

Phase 3 只有同时满足以下条件才算成功：

- Excel 基于模板生成。
- 推荐候选、待人工复核、排除摘要分区正确。
- 原始导出数据完整保留。
- 统计数量与复判 JSON 一致。
- 校验脚本通过。
- 输出路径清晰可追溯。
