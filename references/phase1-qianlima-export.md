# Phase 1 · 千里马两天标讯导出状态机

## 适用范围

用于千里马 VIP 订阅页的每日标讯导出。目标是导出“昨天 00:00:00 至今天 23:59:59”的全部搜索结果 Excel，并确保下载的是本次新生成的文件，不是历史导出。

本阶段是整个工作流最容易卡死的部分。必须按状态机执行，不允许自由发挥。

## 输入

- 本机日期。
- 千里马 VIP 登录态或本机敏感配置中的账号信息。
- 订阅页 URL。
- 浏览器控制能力：正式无人值守 Phase 1 使用专用 CDP Chrome；已有 Kimi 登录时仅按运行包明确策略沿用，不同时强制登录两通道。

## 输出

- 下载成功的千里马 Excel 文件路径。
- 导出记录目标行信息：导出时间、导出条数、导出格式、状态、操作。
- Phase 1 日志，至少包含日期范围、导出配置、导出记录状态变化、下载文件校验结果。

## 不允许的行为

- 不允许使用“今天”“近一周”等快捷日期替代自定义两天范围。
- 不允许点“全选”后按选中内容导出；必须导出“全部搜索结果”。
- 不允许下载旧导出记录。
- 不允许只看第一行就判断成功或失败。
- 不允许清空浏览器数据、删除 Cookie、关闭用户正在使用的浏览器。
- 不允许遇到弹窗就停下，除非弹窗内容无法识别或涉及验证码/安全验证。

## 日期设置规则

1. 读取本机日期。
2. 计算目标范围：
   - 开始：昨天 `00:00:00`
   - 结束：今天 `23:59:59`
3. 进入订阅页。
4. 设置“其他筛选”中的发布时间/查询时间为自定义日期范围。
5. 验证页面显示目标日期范围。
6. 如果页面仍显示“近一周”“今天”“昨天”等快捷项，视为失败，重新设置自定义日期。

## 导出配置规则

点击“批量导出”后，导出配置必须满足：

- 导出范围：全部搜索结果
- 导出方式：Excel
- 导出内容：拓展字段

注意：千里马导出配置的真正导出控件可能不是 `<button>`，可能是 `div.sure-btn` 或自定义元素。查找按钮时使用可见文本和 class 双重判断。

可接受的导出确认文本包括：

- 导出
- 导 出
- 确定
- 确 定

如果点击后出现“已订阅成功”，不要把它当作导出成功。关闭弹窗后进入导出记录页核对。

## 导出数量提示处理

可能出现弹窗：

- 标题：导出数量提示
- 内容可能包含：访问额度不足、是否消耗今日剩余额度、是否继续导出、导出数量限制
- 按钮可能是：继续导出、导 出、导出、确定

处理规则：

1. 只处理最上层可见弹窗。
2. 如果弹窗标题或正文包含“导出数量提示”“访问额度”“继续导出”，点击确认动作。
3. 优先点击红色/主按钮。
4. 如果有“取消”和“继续导出”，必须点击“继续导出”。
5. 点击后等待页面状态变化，不要重复提交多个导出任务。

## 进入导出记录页

进入“我的导出记录”必须通过页面内导航或当前站内正确入口。不要使用旧的猜测 URL。

如果导航后页面为 404，说明入口错误，必须回到 VIP 页面重新通过菜单进入，不要继续猜 URL。

## 目标行选择规则

不要默认第一行是目标行。目标行必须按以下证据综合判断：

1. 导出时间接近本次点击导出的时间，优先选择 15 分钟内记录。
2. 导出条数接近导出前页面显示的总条数。
3. 导出格式为 Excel。
4. 导出内容备注包含当前订阅关键词。
5. 状态和操作列符合导出流程。

当页面同时出现多条新记录时，按以下优先级选择目标行：

1. 时间最接近本次导出的记录。
2. 条数最接近导出前页面总数的记录。
3. 状态和操作只决定下一步动作，不参与目标行身份选择；不得因旧行已有下载而切换目标。
4. 额度弹窗出现后，以弹窗允许数量更新预期条数；目标行首次按本次提交时间和更新后条数锁定，后续不得切换。

禁止行为：

- 不允许因为第一行没有下载就判定失败。
- 不允许下载 6/13、6/12 等历史记录来冒充今天结果。
- 不允许只凭“下载”按钮存在就下载，必须先确认目标行。

## 导出记录状态机

### 状态 A：待生成

页面表现：

- 状态列：待生成
- 操作列：生成

动作：

1. 同一目标行只点击一次“生成”。
2. 如果出现“导出数量提示”，解析剩余额度并只点击一次“继续导出”。
3. 等待 15-20 秒。
4. 刷新并重新读取同一目标行状态；仍为待生成时不得重复提交生成。

### 状态 B：生成中

页面表现：

- 状态列：生成中
- 操作列：刷新

动作：

1. 等待 10-20 秒。
2. 点击目标行“刷新”。
3. 重新读取目标行状态。
4. 最多持续等待到配置的超时时间。

### 状态 C：校验中

页面表现：

- 状态列：校验中
- 操作列：刷新

动作：

1. 等待 10-20 秒。
2. 点击目标行“刷新”。
3. 重新读取目标行状态。
4. 不要点其他历史行的下载。

### 状态 D：已生成

页面表现：

- 状态列：已生成
- 操作列：下载

动作：

1. 再次确认该行是目标行。
2. 如果下载按钮在视口外，先滚动到目标行。
3. 点击目标行“下载”。
4. 检查下载目录中新文件。

### 状态 E：已过期或 0 条

页面表现：

- 状态列：已过期
- 导出条数：0
- 操作列：无下载

动作：

1. 不要下载。
2. 如果这是本次目标行，记录失败并停止。
3. 如果这是历史行，跳过，继续寻找目标行。

## 下载文件校验

下载后必须验证：

1. 文件修改时间晚于本次导出开始时间。
2. 文件大小大于 0。
3. 文件扩展名为 `.xlsx` 或千里马实际 Excel 格式。
4. Excel 行数与导出记录条数接近。
5. 文件不是历史同名旧文件。

如果下载目录出现重名文件，例如 `千里马信息导出_06-15 (2).xlsx`，选择修改时间最新的文件。

## 登录态异常

如果页面出现：

- 登录状态超时，请重新登录
- 请重新登录
- Access Verification
- 账号风险提醒

处理规则：

1. 登录超时：点击“确定”或“确 定”，从 `https://vip.qianlima.com/` 进入登录流程。
2. 账号风险提醒：点击“忽略”，不要修改密码。
3. Access Verification 或验证码：停止，记录截图，要求人工介入。
4. 异地登录或账号被踢：停止，不要反复重试。

## 超时策略

导出记录等待建议：

- 100 条以内：最多等待 3 分钟。
- 100-500 条：最多等待 10-15 分钟。
- 接近 1000 条：最多等待 20 分钟，超过后记录为后台生成超时。

等待期间必须持续记录状态变化，不允许静默卡住。
脚本每 15 秒输出心跳；Agent 只启动一次脚本，不得自行接管网页连续点击。脚本内部最多等待 20 分钟，外层编排器至少给 25 分钟，避免刚到脚本超时时被外层强杀。

每个事件实时写入 `qianlima_mainline_live_*.jsonl`，目标行上下文写入 `qianlima_phase1_checkpoint_*.json`。进程异常退出时，使用 `--resume-context <checkpoint.json>` 继续原目标行，不重新选择日期或再次提交导出。

## 成功标准

Phase 1 只有同时满足以下条件才算成功：

- 自定义日期为昨天至今天。
- 导出配置为全部搜索结果、Excel、拓展字段。
- 导出记录目标行时间和条数匹配本次导出。
- 目标行状态为已生成。
- 已点击目标行下载。
- 本地存在新下载的 Excel 文件。
- 日志写明导出记录目标行。

## 失败输出

如果 Phase 1 失败，必须返回：

- 当前页面 URL。
- 当前截图路径。
- 目标日期范围。
- 是否已提交导出。
- 导出记录页前 5 行状态。
- 最后一次尝试的动作。
- 明确下一步：登录、刷新、生成、继续导出、等待后台、人工验证码，或脚本修复。
