# 已验证基准案例

## 适用范围

用于冷启动 Agent 对照历史成功和失败样本，避免重复犯错。这里不保存完整日志，只保存可复用结论、关键路径、成功标准和禁止回退的规则。

## 使用方式

- 开始修改 Skill 或正式包脚本前，先读本文件。
- 遇到相同症状时，优先采用已验证恢复动作。
- 如果新运行结果与基准案例冲突，先检查是否页面变化或脚本版本过旧。
- 新增稳定成功样本时，追加到本文件，不要覆盖旧案例。

## 基准案例 2026-06-11 · WorkBuddy 全流程跑通

### 背景

WorkBuddy 使用测试运行包复跑四阶段。第一次因执行方式不熟悉停顿，后续按 Kimi WebBridge 完成 Phase 1，并继续 Phase 2-4。

### 关键结果

- Phase 1 下载 Excel：`千里马信息导出_06-11.xlsx`
- 导出记录：约 343 条。
- Phase 2 最终结果：推荐候选 0，待人工复核 4，排除 154。
- Phase 3 Excel：生成成功。
- Phase 4：本地归档、共享盘归档、钉盘上传、钉钉发送成功。

### 已验证结论

- Kimi WebBridge 可用于 Phase 1，不必强依赖 CDP 9222。
- `div.sure-btn` 是导出配置弹窗中的真实导出按钮之一。
- 出现导出数量提示时，需要点击 `继续导出` 或等价主按钮。
- Phase 4 子进程需要清理异常 `NODE_OPTIONS`，否则 DWS CLI 可能失败。

### 不要回退

- 不要因为 CDP 9222 不通就直接判定全流程不可执行。
- 不要让 Agent 每一步都停下来问用户。
- 不要发没有钉盘链接的钉钉消息。

## 基准案例 2026-06-13 · 自动任务失败样本

### 背景

QClaw 定时任务早上执行两次失败。

### 观察

第一次：

- 登录态过期。
- 出现“登录状态超时，请重新登录”。
- 日期筛选无法设置。

第二次：

- 登录态恢复。
- 日期筛选成功。
- 导出配置成功。
- 导出记录出现约 350 条，状态为 `待生成`。
- Agent 反复点击 `生成`，最终超时。

### 已验证问题

- 登录态不能用弱特征判断。
- 导出记录页的 `待生成` 不是失败，必须进入生成/继续导出/刷新状态机。
- 几百条导出可能需要较长后台校验时间。

### 应用到 Skill 的规则

- 登录超时必须恢复登录后继续，不能直接失败。
- `待生成` 必须点击目标行 `生成`。
- 弹出导出数量提示时点击 `继续导出`。
- `生成中/校验中` 必须等待并点击目标行 `刷新`。

## 基准案例 2026-06-15 · Phase 1 下载页卡点复盘

### 背景

用户手动截图确认千里马导出记录页的真实状态变化。当天导出记录页出现多种状态和多条记录。

### 页面状态

观察到的行状态包括：

- `校验中` + 操作 `刷新`
- `待生成` + 操作 `生成`
- 弹窗 `导出数量提示`
- 弹窗按钮 `继续导出`
- `生成中` + 操作 `刷新`
- `已生成` + 操作 `下载`

还观察到：

- 第一行可能是 1000 条 `校验中`。
- 第二行可能是 349/350 条 `已生成` 且可下载。
- 不能只处理第一行。

### 已验证结论

- 目标行必须按时间、条数、状态、操作列选择。
- 如果第一行不是目标行，必须扫描后续行。
- 目标行从 `待生成` 到 `下载` 的状态变化是正常流程，不是异常。
- 弹窗有时出现，有时不出现；脚本必须兼容两种情况。

### 历史记录说明

2026-06-15 曾做过一版 Phase 1 脚本修复，内容包括登录超时文本归一化、导出记录目标行选择、禁用旧 URL 猜测、下载前滚动并重取按钮坐标。

这只是历史修复记录，不代表当前有效脚本版本。2026-06-17 已完成真实网页 Phase 1 状态机测试，应以本文件顶部的 2026-06-17 基准为准。

## 筛选规则基准

### 已确认容易误判的排除项

- 选聘资产评估。
- 值班用房租赁。
- 检验检测、监测。
- 纯咨询设计。
- 纯道路硬化或路面拓宽，无照明/电气范围。
- 纯设备采购，无安装施工。

### 已确认需要待人工复核的边界项

- 城中村改造，正文或附件可能含亮化/电气。
- 学校或医院小额灯具更换。
- 综合改造项目，正文不足但附件可预览。
- 光伏标题中出现租赁、评估、服务时，需要区分施工和非施工。
- 充电桩标题中出现系统、设备、配件时，需要区分安装施工和纯采购。

## 基准案例 2026-06-18 · CDP 读取器历史测试不能直接视为生产验收

### 背景

历史测试包曾报告 DOCX/PDF 下载成功和“6/6 验收通过”。

### 复核结论

- 测试证明千里马两步下载机制真实存在，DOCX 和部分 PDF 可以取得真实文件。
- 当时批量正文成功率约 80%，有附件项目完整读取率约 50%，并非全量成功。
- 扫描 PDF 只提取到 4 或 20 个字符，却被历史脚本标为成功。
- 历史脚本会清空附件下载目录、复用首个浏览器标签页、部分附件成功即标整个项目成功。
- 因此该版本只能作为技术探索，不是当前生产基准。

### 不要回退

- 不要引用“6/6 通过”证明正式流程已完整。
- 不要把短文本 PDF 当有效证据。
- 不要调用历史 v02/v03 读取器。
