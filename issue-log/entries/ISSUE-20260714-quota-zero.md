## 1. 遇到什么问题

弹窗「访问额度不足…今日剩余额度 0 条」后点继续导出，目标行长期「待生成」，无「下载」，空刷十余分钟。

## 2. 影响范围

- 阻断：正式新导出 xlsx，无法纯 Phase1→34  
- 联调：可用已生成行 fallback 续 2–4  

## 3. 怎么解决的

- S1：解析剩余额度 0 → fail-fast（~25s）  
- S5：`--allow-existing-export-download`（默认关）可下载已生成行  

## 4. 根因

账号导出额度耗尽（站点侧），非按钮状态机逻辑错误。

## 5. 后续重点改什么

- [ ] 额度恢复后实机验证 fail-fast 与正常下载  
- [ ] 钉钉失败文案是否要对用户更友好
